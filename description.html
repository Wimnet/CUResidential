About Methodology Data Description and Sharing

Data Description

On the server side, our system captures network traffic using Gulp (in pcap format) and extracts key fields with tshark, including TLS Server Name Indication (SNI) and DNS A records. For domain names, we retain only the last three components (e.g., from a.b.c.d, we keep b.c.d) to preserve relevant information while reducing granularity.

We store the extracted fields in CSV files and then anonymize sensitive data:

  • IP Addresses:
    • External IPs (outside the university): kept as-is.
    • IPs (within student housing): fully anonymized using the Crypto-PAn scheme. However, we retain subnet-level information to associate traffic with specific buildings and units—without revealing exact IPs.
    • Non-residential university IPs: reduced to their university subnets.
  • MAC Addresses: We anonymize all except the Vendor ID, which is preserved.
Payload data beyond Layer 4 is removed using DagScrubber to further protect privacy. Once anonymized, the pipeline uploads the processed pcap and CSV files to the cloud and securely deletes the original raw data.

Access Request Terms

We are authorized by Columbia University’s IT and Institutional Review Board (IRB) to share our datasets with external researchers.

To ensure user privacy and prevent any risk of de-anonymization, we will share the dataset only with researchers who:

  • Have IRB approval or exemption from their own institution, and
  • Agree to our Acceptable Use Policy (see Appendix A.1.2 for details)
